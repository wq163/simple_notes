## Bug 1：从分类列表新建笔记，不会默认分配给当前分类

**问题描述**：从侧栏进入某个分类后，点击「新建笔记」，编辑器的分类选择器总是默认选中「默认分类」，而非当前浏览的分类。

**根本原因**：`NoteEditorView.vue` 在 `onMounted` 中初始化分类时，总是取 `defaultCategory`，没有考虑当前路由的分类上下文。

### 改动文件

#### `src/client/views/NoteEditorView.vue`

```diff
-  // Set default category
-  const defaultCat = notesStore.categories.find(c => c.isDefault);
-  if (defaultCat) selectedCategory.value = defaultCat.id;
+  // Set default category: 如果当前在某个分类页面，新建笔记默认使用该分类
+  if (!isEditing.value && route.name === 'Category' && route.params.id) {
+    selectedCategory.value = route.params.id as string;
+  } else {
+    const defaultCat = notesStore.categories.find(c => c.isDefault);
+    if (defaultCat) selectedCategory.value = defaultCat.id;
+  }
```

## Bug 2：新建笔记保存后，列表跳到首页分类

**问题描述**：在某个分类下新建笔记并保存后，左侧笔记列表变成了首页的全部笔记，而非保留在当前分类视图。

**根本原因**：`saveNote` 中保存后调用 `await notesStore.fetchNotes()` 没有传参数，总是拉取全部笔记，覆盖了 store 中的列表数据。

### 改动文件

#### `src/client/views/NoteEditorView.vue`

```diff
-    await notesStore.fetchNotes();
+    // 根据当前路由上下文刷新笔记列表，避免跳到首页
+    if (route.name === 'Category') {
+      await notesStore.fetchNotes({ categoryId: route.params.id as string });
+    } else if (route.name === 'Tag') {
+      await notesStore.fetchNotes({ tagId: route.params.id as string });
+    } else if (route.name === 'Search') {
+      await notesStore.fetchNotes({ search: route.query.q as string });
+    } else {
+      await notesStore.fetchNotes();
+    }
```

## Bug 3：截图后不能直接 Ctrl+V 粘贴到笔记中

**问题描述**：使用截图工具（如 Win+Shift+S）截图后，在编辑器中按 Ctrl+V 无法粘贴图片。

**根本原因**：Milkdown 编辑器没有注册 `handlePaste` 处理器来拦截剪贴板中的图片数据。只有通过工具栏按钮选择文件才能上传图片。

### 改动文件

#### `src/client/components/MilkdownEditorInner.vue`

新增 ProseMirror 插件，通过 `$prose` 注入一个 `handlePaste` 处理器：

- 检测 `clipboardData.items` 中的图片类型文件
- 调用已有的 `fileUpload` / `imageUpload` 上传图片
- 上传完成后通过 `insertMarkdown` 插入图片链接
- 返回 `true` 阻止默认粘贴行为

```diff
+import { $prose } from '@milkdown/utils';
+import { Plugin, PluginKey } from '@milkdown/prose/state';

+  // Register clipboard paste handler for images
+  const pasteUploadPlugin = $prose(() => new Plugin({
+    key: new PluginKey('clipboard-image-paste'),
+    props: {
+      handlePaste(view, event) {
+        // 检查剪贴板中是否有图片文件
+        // 有则上传并插入 markdown 图片链接
+        // 返回 true 阻止默认粘贴
+      },
+    },
+  }));
+  crepe.editor.use(pasteUploadPlugin);
```

## Bug 4：笔记列表下拉后点击某篇笔记，列表滚回顶部

**问题描述**：在笔记列表中向下滚动后，点击某篇笔记打开编辑器，左侧列表的滚动位置被重置到顶部。

**根本原因**：路由 query 参数变化时，浏览器的 History API 默认自动管理滚动位置（`history.scrollRestoration = 'auto'`），在 `pushState` 时异步重置了页面内滚动容器的 `scrollTop`。同时 `.workspace-view` 的高度设置不当导致滚动容器层级混乱。

### 改动文件

#### `src/client/router/index.ts`

添加 `scrollBehavior` 配置，让 Vue Router 设置 `history.scrollRestoration = 'manual'`，阻止浏览器自动管理导航时的滚动位置：

```diff
 const router = createRouter({
   history: createWebHistory(configBase),
+  scrollBehavior() {
+    return false;
+  },
   routes: [...]
 });
```

#### `src/client/views/HomeView.vue`

**CSS 修复**：将 `.workspace-view` 高度从 `calc(100vh - 80px)` 改为 `100%`，添加 `overflow: hidden`：

```diff
 .workspace-view {
   display: flex;
-  height: calc(100vh - 80px);
+  height: 100%;
+  overflow: hidden;
   gap: var(--spacing-md);
 }
```

**JS 修复**：给 `.notes-list-pane` 添加 `ref`，在 `openNote` 中保存滚动位置，导航后用 `requestAnimationFrame` 轮询机制（30帧/~500ms）持续恢复滚动位置，覆盖浏览器的任何异步重置：

```diff
+const notesListPaneRef = ref<HTMLElement>();

-function openNote(note: NoteListItem) {
+async function openNote(note: NoteListItem) {
   if (isTrash.value) return;
+  const pane = notesListPaneRef.value;
+  const contentArea = pane?.closest('.content-area') as HTMLElement | null;
+  const paneScroll = pane?.scrollTop ?? 0;
+  const contentScroll = contentArea?.scrollTop ?? 0;
+  await router.push({ ... });
+  await nextTick();
+  let frameCount = 0;
+  const restoreScroll = () => {
+    if (frameCount >= 30) return;
+    frameCount++;
+    if (pane && pane.scrollTop !== paneScroll) pane.scrollTop = paneScroll;
+    if (contentArea && contentArea.scrollTop !== contentScroll) contentArea.scrollTop = contentScroll;
+    requestAnimationFrame(restoreScroll);
+  };
+  restoreScroll();
 }
```
