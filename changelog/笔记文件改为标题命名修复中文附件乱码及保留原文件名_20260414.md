# 笔记文件改为标题命名、修复中文附件乱码及保留原文件名

**时间**：2026-04-14

---

## 需求概述

1. 笔记文件名由 UUID 改为以笔记第一行（标题）命名
2. 附件为中文文件名时，上传后笔记中显示的名称出现乱码，需修复
3. 附件和图片上传后磁盘存储保留原文件名，不再用 UUID 重命名

---

## 改动文件

### `src/server/routes/notes.ts`

**新增辅助函数：**

- `titleToFilename(content)` —— 提取 Markdown 内容第一行，去除 `#` 标题前缀，过滤文件系统非法字符（`/ \ : * ? " < > |`），限制最长 80 字符，空标题回退为 `无标题笔记`
- `resolveUniqueFilePath(dir, baseName, excludeCurrent?)` —— 生成唯一文件名，冲突时自动追加 `-1`、`-2` 等后缀；`excludeCurrent` 用于更新场景跳过当前文件自身

**POST 创建笔记：**

- 不再以 UUID 作为文件名，改为调用 `titleToFilename` + `resolveUniqueFilePath` 生成 `<标题>.md`
- UUID 仍用于数据库主键 `id`

**PUT 更新笔记：**

- 每次保存时根据新内容重新计算文件名
- 若文件名发生变化（标题改变），执行 `fs.writeFileSync` 写新文件 + `fs.unlinkSync` 删除旧文件 + `UPDATE notes SET file_path = ?` 同步数据库

---

### `src/server/routes/attachments.ts`

**移除：**

- 删除内存 `fileNameMap`（服务重启后本就会丢失，改为保留原文件名后彻底不需要）
- 删除 `uuid` 依赖（文件名不再重命名）

**新增辅助函数：**

- `decodeOriginalName(raw)` —— 修复中文乱码：`multer` 从 HTTP multipart header 读取的 `file.originalname` 有时被错误解释为 Latin-1，通过 `Buffer.from(raw, 'latin1').toString('utf8')` 重新解码，包含非 ASCII 字符时才替换
- `sanitizeFilename(name)` —— 去除路径遍历字符（`/ \ : * ? " < > |` 及 `\x00`），保留中文等 Unicode 字符
- `resolveUniqueFilename(dir, filename)` —— 同名文件存在时自动追加 `-1`、`-2` 后缀

**`multer` `filename` 回调：**

- 解码原始文件名 → 清理非法字符 → 解决冲突后，直接保存到磁盘，文件名与用户上传文件一致

**上传结果 `url` 字段：**

- 对文件名做 `encodeURIComponent` 编码，确保中文文件名在 URL 中合法

**`publicAttachmentHandler`：**

- 对 URL 参数中的 `filename` 先做 `decodeURIComponent`，再 `path.basename` 防路径遍历
- 下载模式下，`Content-Disposition` 使用 `filename*=UTF-8''...` 格式确保中文文件名在浏览器中正确展示

---

## 注意事项

- **历史数据兼容**：已有笔记的 `file_path` 仍为旧 UUID 命名，无需迁移，系统照常读取
- **已有附件**：旧 UUID 命名的附件文件不变，已嵌入笔记的链接仍然有效
- **文件名冲突**：同用户下同标题笔记 / 同名附件会自动追加数字后缀
