# 修复附件图片链接在baseUrl子路径部署下失效的问题

**时间**：2026-04-14

---

## 问题描述

当 `config/default.json` 中配置了 `baseUrl`（例如 `/notes`）时，笔记内嵌入的附件和图片链接会失效。

**根本原因**：上传附件后，服务器返回的 URL 以 `/` 开头（如 `/api/attachments/user/file.jpg`），这是绝对路径。当应用部署在子路径 `/notes/` 下时，实际的 API 地址是 `/notes/api/attachments/...`，绝对路径无法匹配。

---

## 改动文件

### `src/server/routes/attachments.ts`

**改动位置**：`POST /api/attachments` 路由的上传响应中，`url` 字段的拼接方式

```diff
- url: `/api/attachments/${username}/${encodeURIComponent(file.filename)}`,
+ // Use a relative URL (no leading slash) so it works under any baseUrl configuration
+ url: `api/attachments/${username}/${encodeURIComponent(file.filename)}`,
```

**原理**：去掉开头的 `/`，改为相对路径。浏览器在渲染 Markdown 中的图片/链接时，会将相对路径解析为相对于当前页面 URL 的路径：

- 无 baseUrl（部署在 `/`）：`api/attachments/...` → 解析为 `/api/attachments/...` ✓
- `baseUrl=/notes`（部署在 `/notes/`）：`api/attachments/...` → 解析为 `/notes/api/attachments/...` ✓

---

## 无副作用说明

- 下载链接参数 `?download=1` 不受影响
- 文件名 URL encode 逻辑不变
- 已存入笔记的旧绝对路径链接不受影响（历史数据无需迁移）
